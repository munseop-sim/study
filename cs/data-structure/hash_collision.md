# 해시 충돌 (Hash Collision)

> 출처: https://www.maeil-mail.kr/question/210
> 기존 파일 `HashTable.md`에 해시 테이블 기본 구조 참고

---

## 개념

**해시(Hash) 자료구조**는 키-값 쌍으로 이루어진 데이터 구조로, 키를 이용해 값을 O(1) 시간 복잡도로 찾을 수 있습니다. 키를 해시 함수에 넣어서 나오는 결과(해시코드)를 기반으로 인덱스를 생성하고 해당 위치에 값을 저장합니다.

**해시 충돌(Hash Collision)** 이란 서로 다른 키를 해시 함수에 넣었을 때 **같은 결과(해시코드)**가 나오는 경우를 말합니다.

---

## 해시 충돌 해결 방법

해시 충돌을 완화하기 위한 대표적인 두 가지 접근 방법이 있습니다.

### 1. 분리 연결법 (Separate Chaining)

버킷을 **연결 리스트나 트리 형태**로 관리하여 버킷에 들어갈 값의 수에 제한을 두지 않는 방법입니다.

```
버킷[0] → [K1:V1] → [K2:V2] → null
버킷[1] → [K3:V3] → null
버킷[2] → [K4:V4] → [K5:V5] → [K6:V6] → null
```

**특징:**
- 별도의 연결 리스트 또는 트리 구조 사용
- 버킷에 저장되는 값의 수에 제한 없음
- 메모리 사용량이 상황에 따라 동적으로 변함

**Java의 HashMap 구현:**
- Java 8 이후, 체인의 항목 개수가 임계치(기본값: 8)를 초과하면 연결 리스트 대신 **Red-Black Tree**를 사용하여 탐색 시간을 O(log n)으로 줄임

---

### 2. 개방 주소법 (Open Addressing)

특정 값이 들어가야 하는 자리(버킷)가 이미 사용되고 있는 경우, **다른 해시 버킷에 데이터를 삽입**하는 방법입니다. 하나의 해시 테이블만 사용하며 별도의 추가 구조가 필요 없습니다.

#### 2-1. 선형 탐사법 (Linear Probing)

임의의 고정된 크기만큼 **한 칸씩 순차적으로 이동**하면서 빈 버킷을 찾는 방법입니다.

```
충돌 발생 → +1, +2, +3, ... 순서로 탐색
index = (hash(key) + i) % capacity
```

**단점:** 특정 버킷 주변이 모두 채워져 있는 경우 성능 저하 → **1차 군집 현상(Primary Clustering)**

해시 충돌이 테이블 전체에 균등하게 발생할 때 유용합니다.

#### 2-2. 제곱 탐사법 (Quadratic Probing)

선형 탐사법처럼 한 칸씩 찾는 것이 아닌 **제곱으로 늘리면서** 빈 버킷을 찾는 방법입니다.

```
index = (hash(key) + i^2) % capacity
충돌 발생 → +1, +4, +9, +16, ... 순서로 탐색
```

**장점:** 보폭이 점점 늘어나기 때문에 특정 영역에 값이 밀집되어도 빠르게 벗어날 수 있음

**단점:** 여러 값이 해시 함수로 같은 값을 갖게 될 경우 모두 같은 순서로 탐사 → **2차 군집 현상(Secondary Clustering)**

#### 2-3. 이중 해싱 (Double Hashing)

해시 충돌이 발생하는 경우, **보조 해시 함수**를 사용하여 다음 버킷을 결정하는 방법입니다.

```
index = (hash1(key) + i * hash2(key)) % capacity
```

**장점:** 해시 충돌 가능성이 가장 낮음 (군집 현상 최소화)

**단점:** 추가적인 보조 해시 함수 연산이 발생하여 다른 방식에 비해 연산량이 더 많음

## 세 가지 방법 비교

| 방법 | 군집 현상 | 메모리 | 충돌 해결 능력 | 연산량 |
|---|---|---|---|---|
| 선형 탐사법 | 1차 군집 | 낮음 | 낮음 | 낮음 |
| 제곱 탐사법 | 2차 군집 | 낮음 | 중간 | 중간 |
| 이중 해싱 | 없음 | 낮음 | 높음 | 높음 |
| 분리 연결법 | 없음 | 높음 | 높음 | 중간 |
