# Operating System Basics

운영체제 관점의 자원 관리 기초를 정리한 문서입니다.

## 스레드, 프로세스, 코어의 수는 많을수록 좋을까?
- 수를 늘리면 병렬성은 좋아질 수 있지만, 무조건 성능이 좋아지지는 않습니다.

### 스레드가 많으면?
- 컨텍스트 스위칭 비용이 증가해 효율이 떨어질 수 있습니다.
- 메모리, 캐시, 락 경합으로 병목이나 데드락 위험이 커질 수 있습니다.

### 프로세스가 많으면?
- 독립 메모리 사용으로 메모리 사용량이 크게 늘어날 수 있습니다.
- IPC 비용이 커져 성능 저하가 발생할 수 있습니다.

### 코어가 많으면?
- 멀티코어 최적화된 소프트웨어에서는 이점이 큽니다.
- 단일 스레드 위주 작업에서는 효과가 제한적입니다.
- 전력 소비, 발열, 하드웨어 비용을 함께 고려해야 합니다.
